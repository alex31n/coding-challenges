# 15. 3Sum

**Difficulty:** `Medium`

**Topics :** `Array` `Two Pointers` `Sorting`

**Source:** [LeetCode - 3Sum](https://leetcode.com/problems/3sum/)

---

Given an integer array `nums`, return all the triplets `[nums[i], nums[j], nums[k]]` such that `i != j`, `i != k`, and `j != k`, and `nums[i] + nums[j] + nums[k] == 0`.

Notice that the solution set must not contain duplicate triplets.

**Example 1:**  
> Input: `nums = [-1,0,1,2,-1,-4]`  
> Output: `[[-1,-1,2],[-1,0,1]]`  
> Explanation: 
> `nums[0] + nums[1] + nums[2] = (-1) + 0 + 1 = 0.`  
> `nums[1] + nums[2] + nums[4] = 0 + 1 + (-1) = 0.`  
> `nums[0] + nums[3] + nums[4] = (-1) + 2 + (-1) = 0.`  
> `The distinct triplets are [-1,0,1] and [-1,-1,2].`  
> `Notice that the order of the output and the order of the triplets does not matter.`  

**Example 2:**  
> Input: `nums = [0,1,1]`  
> Output: `[]`  
> Explanation: `The only possible triplet does not sum up to 0.`  

**Example 3:**  
> Input: `nums = [0,0,0]`  
> Output: `[[0,0,0]]`  
> Explanation: `The only possible triplet sums up to 0.`  

**Constraints:**
- `3 <= nums.length <= 3000`
- `-10^5 <= nums[i] <= 10^5`

## Solution
The problem asks us to find all unique combinations of three numbers in an array that sum up to zero. The main challenges are efficiently finding these triplets and handling duplicates to ensure the final output contains only unique sets.

**Goal:** Find all unique triplets `[a, b, c]` in the array `nums` such that `a + b + c = 0`.

### 1. Two Pointers with Sorting (Optimal)
This is the most efficient and common approach. By first sorting the array, we can use a two-pointer technique to find the remaining two numbers for each element, and the sorted nature helps us easily skip duplicate values.

**Business Logic:**
- **Sort the array:** This is crucial. It allows us to use the two-pointer method and easily skip duplicates.
- **Iterate with the first pointer:** Loop through the array with an index `i` from the start. This `nums[i]` will be the first number in our potential triplet.
- **Skip duplicates:** If the current `nums[i]` is the same as the previous one, we skip it to avoid generating duplicate triplets.
- **Use two more pointers:** For each `nums[i]`, initialize a left pointer `l` at `i + 1` and a right pointer `r` at the end of the array.
- **Find the sum:**
    - If `nums[i] + nums[l] + nums[r]` is greater than 0, we need a smaller sum, so we decrement `r`.
    - If the sum is less than 0, we need a larger sum, so we increment `l`.
    - If the sum is exactly 0, we've found a valid triplet. We add it to our results. Then, we must move our `l` and `r` pointers while also skipping any duplicates they might point to.

### Complexity
- Time Complexity: $O(n^2)$. Sorting the array takes $O(n \log n)$. The main loop iterates `n` times, and the two-pointer `while` loop inside takes $O(n)$ in the worst case, leading to a total of $O(n^2)$.
- Space Complexity: $O(1)$ or $O(n)$, depending on the space complexity of the sorting algorithm used. The space for the output array is not counted.

### Implementation
Python
```python
class Solution:
    def threeSum(self, nums: List[int]) -> List[List[int]]:
        res = []
        nums.sort()

        for i, a in enumerate(nums):
            # If the first number is positive, the sum can't be zero.
            if a > 0:
                break
            # Avoid duplicate triplets by skipping the same starting number.
            if i > 0 and a == nums[i - 1]:
                continue

            l, r = i + 1, len(nums) - 1
            while l < r:
                threeSum = a + nums[l] + nums[r]
                if threeSum > 0:
                    r -= 1
                elif threeSum < 0:
                    l += 1
                else:
                    res.append([a, nums[l], nums[r]])
                    l += 1
                    r -= 1
                    # Move left pointer forward to skip duplicates of the next number.
                    while l < r and nums[l] == nums[l - 1]:
                        l += 1
                    # Move right pointer backward to skip duplicates of the previous number.
                    while l < r and nums[r] == nums[r + 1]:
                        r -= 1
        return res
```
Java
```java
import java.util.ArrayList;
import java.util.Arrays;
import java.util.List;

class Solution {
    public List<List<Integer>> threeSum(int[] nums) {
        List<List<Integer>> res = new ArrayList<>();
        Arrays.sort(nums);

        for (int i = 0; i < nums.length - 2; i++) {
            // If the first number is positive, the sum can't be zero.
            if (nums[i] > 0) {
                break;
            }
            // Avoid duplicate triplets by skipping the same starting number.
            if (i > 0 && nums[i] == nums[i - 1]) {
                continue;
            }

            int l = i + 1;
            int r = nums.length - 1;
            while (l < r) {
                int threeSum = nums[i] + nums[l] + nums[r];
                if (threeSum > 0) {
                    r--;
                } else if (threeSum < 0) {
                    l++;
                } else {
                    res.add(Arrays.asList(nums[i], nums[l], nums[r]));
                    l++;
                    r--;
                    // Move left pointer forward to skip duplicates of the next number.
                    while (l < r && nums[l] == nums[l - 1]) {
                        l++;
                    }
                    // Move right pointer backward to skip duplicates of the previous number.
                    while (l < r && nums[r] == nums[r + 1]) {
                        r--;
                    }
                }
            }
        }
        return res;
    }
}
```
